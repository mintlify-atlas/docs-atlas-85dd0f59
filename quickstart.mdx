---
title: "Quickstart: Install and connect Cruncher MCP in minutes"
description: "Connect Cruncher to your AI client in minutes — no local install needed. This guide walks you from prerequisites to your first math calculation."
---

Cruncher MCP is distributed as an npm package (`@slbdn/cruncher-mcp`) and runs as a local server your AI client connects to over stdio. This guide walks you through getting Cruncher running with Claude Desktop — the most common setup. For other clients, see [Connect Cruncher to your AI client](/clients).

<Steps>
  <Step title="Check prerequisites">
    Cruncher requires **Node.js 18 or newer**. To check your current version, run:

    ```bash
    node --version
    ```

    If you see `v18.x.x` or higher, you're ready. If not, download Node.js from [nodejs.org](https://nodejs.org/).
  </Step>

  <Step title="Run Cruncher via npx">
    You can verify the server starts correctly before configuring any client:

    ```bash
    npx @slbdn/cruncher-mcp
    ```

    The server will start and wait for an MCP client to connect over stdio. Press `Ctrl+C` to stop it — you don't need to keep it running manually. Your AI client will start and stop Cruncher automatically once configured.

    <Note>
      `npx` downloads and runs the latest version of Cruncher without installing it globally. Run this once to confirm it works, then configure your client below.
    </Note>
  </Step>

  <Step title="Configure Claude Desktop">
    Locate your Claude Desktop configuration file:

    - **macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`
    - **Windows**: `%APPDATA%\Claude\claude_desktop_config.json`

    Add the Cruncher server entry to the `mcpServers` object:

    ```json
    {
      "mcpServers": {
        "cruncher": {
          "command": "npx",
          "args": ["@slbdn/cruncher-mcp"]
        }
      }
    }
    ```

    Save the file, then **completely quit and restart Claude Desktop**. Claude must be fully restarted — not just the window — to pick up changes to the config file.

    <Tip>
      If you'd rather not use `npx`, you can download `cruncher.js` directly from the GitHub repository and point to it with `"command": "node"` and `"args": ["/path/to/cruncher.js"]`. See [Connect Cruncher to your AI client](/clients) for that config.
    </Tip>
  </Step>

  <Step title="Start asking math questions">
    Once Claude Desktop restarts, Cruncher is connected. Try asking:

    - "What is 15% of 240?"
    - "Calculate the average, median, and range of [15, 22, 8, 41, 19, 30]"
    - "What is 2 raised to the power of 10?"
    - "What is the standard deviation of [10, 12, 8, 14, 6]?"
    - "Evaluate the expression: sin(pi/4) + sqrt(16) + log10(100)"
    - "What is 10 factorial?"
    - "Convert 50 miles to kilometers"
    - "A stock went from $50 to $75. What's the percentage change?"

    Claude will call Cruncher's tools automatically — you don't need to mention Cruncher by name.
  </Step>
</Steps>

## Unlock more tools

By default, Cruncher runs in `standard` mode with 34 tools. To enable all 43 tools — including memory, batch processing, base conversion, and percentile calculations — set the `CRUNCHER_TOOL_SET` environment variable to `full`:

```json
{
  "mcpServers": {
    "cruncher": {
      "command": "npx",
      "args": ["@slbdn/cruncher-mcp"],
      "env": {
        "CRUNCHER_TOOL_SET": "full"
      }
    }
  }
}
```

With `full` mode active, you can ask things like:

- "Store 99 in memory, add 5, then recall the total."
- "What's the 75th percentile of [10, 20, 30, 40, 50]?"
- "Convert 11010 from binary to decimal."
- "Batch: square root of 144, add 10, then raise to power 2."
