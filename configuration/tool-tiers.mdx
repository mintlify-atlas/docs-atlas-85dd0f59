---
title: "Cruncher tool tiers: minimal, standard, and full"
description: "Choose the right Cruncher tool tier to balance token budget against capability ‚Äî from 5 tools and ~160 tokens to 43 tools and ~1,500 tokens."
---

Cruncher exposes its 43 tools across three tiers, controlled by the `CRUNCHER_TOOL_SET` environment variable. Each tier is a strict superset of the one before it. Picking a smaller tier reduces the number of tool definitions sent to the model on every request, which directly lowers your context token usage.

## Tier overview

| Tier | Tools | Token budget | Best for |
|------|------:|-------------:|----------|
| `minimal` | 5 | ~160 tokens | Lightweight agents, simple arithmetic, token-constrained contexts |
| `standard` | 34 | ~1,150 tokens | General-purpose AI assistants ‚Äî trig, stats, percentages, constants, unit conversion |
| `full` | 43 | ~1,500 tokens | Power users and multi-step workflows needing memory, base conversion, percentile, batch, and cache |

The default tier is `standard`. You do not need to set `CRUNCHER_TOOL_SET` unless you want a different tier.

<Tip>
  In `minimal` mode, `evaluate_expression` handles all complex math ‚Äî you can still compute `sin(pi/6) + sqrt(16)` as a single expression string. Dropping from `standard` to `minimal` saves up to 90% of the context tokens Cruncher uses.
</Tip>

## Minimal tier ‚Äî 5 tools

## Choosing a tier

Use this guidance to pick the right tier for your setup:

- **minimal** ‚Äî you have a tight token budget, your agent only needs basic arithmetic, or you want to route all complex math through `evaluate_expression` yourself.
- **standard** ‚Äî the right choice for most users. Covers everyday math including trig, statistics, unit conversion, and percentages without a large token cost.
- **full** ‚Äî you need persistent memory across calls, base conversion, percentile calculations, or want to run batches of operations in a single request.

## Setting the tier in your MCP config

Pass `CRUNCHER_TOOL_SET` in the `env` field of your server entry:

```json
{
  "mcpServers": {
    "cruncher-minimal": {
      "command": "node",
      "args": ["/path/to/cruncher.js"],
      "env": { "CRUNCHER_TOOL_SET": "minimal" }
    }
  }
}
```

You can register Cruncher multiple times under different names if you want to switch tiers without editing the config:

```json
{
  "mcpServers": {
    "cruncher": {
      "command": "node",
      "args": ["/path/to/cruncher.js"],
      "env": { "CRUNCHER_TOOL_SET": "standard" }
    },
    "cruncher-full": {
      "command": "node",
      "args": ["/path/to/cruncher.js"],
      "env": { "CRUNCHER_TOOL_SET": "full" }
    }
  }
}
```

## Fuzzy tool name matching

If your model calls a tool name with a typo, Cruncher uses Levenshtein distance to find the closest match and returns a suggestion instead of a generic error.

| Typo called | Suggestion returned |
|-------------|---------------------|
| `sinn` | `sine` |
| `squrt` | `sqrt` |
| `fctorial` | `factorial` |
| `addd` | `add` |
| `totally_wrong` | *(no suggestion ‚Äî too different)* |

The error response includes a human-readable "Did you mean" message:

```json
{
  "error": {
    "code": -32601,
    "message": "Tool 'sinn' not found. Did you mean 'sine'?"
  }
}
```

<Info>
  Fuzzy matching applies across all tiers. If the model calls a tool that exists but is not registered in the active tier (for example, calling `memory_recall` while in `standard` mode), Cruncher returns a `-32601` error and suggests switching to `CRUNCHER_TOOL_SET=full`.
</Info>
